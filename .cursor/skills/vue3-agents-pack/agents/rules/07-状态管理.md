# 状态管理规范（NON-NEGOTIABLE）

## 基础规范

- 必须使用 Pinia，所有 store 放在 `src/stores/`。
- 禁止在 src 下新建其它目录存放 store。

## Store 组织

- 按业务模块划分，每模块单独文件（如 `stores/tape.ts`、`stores/userInfo.ts`）。
- 命名：`use{ModuleName}Store` 或 `useXxxStore`（如 useTapeStore、useUserStore）。
- Store 只存数据与业务行为，不耦合 UI 组件。
- 需要持久化时使用 `persist: true` 或项目采用的 pinia-plugin-persistedstate 等方案。

## 定义方式

- 使用 `defineStore` 定义；state、actions 等按 Pinia 约定。
- 导出 store 实例或通过 `useXxxStore()` 使用。
