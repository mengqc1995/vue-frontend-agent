# 编码规范

## TypeScript 规范

- 所有代码必须使用 TypeScript，禁止使用 JavaScript。
- 禁止使用 `any`（除非有明确理由并注释）。
- 接口、类型使用 PascalCase；组件 Props 必须定义清晰接口。

## 命名规范

### 变量与常量

| 类型 | 规则 | 示例 |
|------|------|------|
| 普通变量/函数 | camelCase | userList, onSubmit, handleQuery |
| 常量 | UPPER_SNAKE_CASE | API_TIMEOUT, MAX_RETRY_COUNT |
| 布尔值变量 | 以 is/has/can/should/will/did 等前缀 | isLoading, hasPermission, canEdit |
| 接口/类型 | PascalCase | UserInfo, TableConfig |

布尔值变量禁止使用无前缀命名（如 `loading`、`permission`），必须用 `isLoading`、`hasPermission` 等。

### 函数命名

- 函数名必须以**动词开头**，使用 camelCase。
- 事件处理：onXxx（如 onSubmit、onChange）。
- 内部处理：handleXxx（如 handleDelete、handleQuery）。
- 常用动词前缀：get/fetch/load、set/update/save、create/add、delete/remove、check/validate、handle/process、format/transform。

禁止函数名仅为名词（如 `query`、`list`、`form`）。

### 目录与文件命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 目录（业务/视图） | camelCase | userManagement, tapeManagement |
| Vue 组件文件 | PascalCase | AddCvrDialog.vue, BaseDialog.vue |
| 路由 path/文件夹 | 与 views 路径对应，见 06-路由规范 | |

### React/Vue 组件与 Hooks

- Vue 组件：PascalCase。
- 自定义 Hooks：use 开头，camelCase（如 useUserStore、useTapeStore 来自 Pinia）。

## 业务函数命名

- 事件处理：onXxx（如 onSubmit）。
- 内部处理：handleXxx（如 handleDelete）。
