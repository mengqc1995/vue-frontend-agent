# 路由规范（NON-NEGOTIABLE）

## 路由结构

- 页面组件位于 `views/` 下，与路由 path 对应（如 path `/HomeView` 对应 `views/homeView/index.vue`）。
- 路由定义在 `router/routes/` 下：
  - 公共路由：`router/routes/common.ts`
  - USM 专属：`router/routes/usm-only.ts`
  - 其它模式按项目约定增加对应文件。
- 建议 path 与文件夹名称对应，便于从浏览器地址定位文件。

## 路由与菜单

- 禁止多处维护路由：全局唯一路由配置，在 `router/routes/` 中集中管理。
- 同一页面只在一处被 import；通过 `getRoutesByMode()` 等按构建模式合并。
- meta 字段（title、icon、roles、order 等）与菜单展示一致，见 `router/route.ts` 中 meta 说明。

## 添加新页面

- 在 `views/` 下新建目录及 `index.vue`（及可选 scss）。
- 在对应 `router/routes/common.ts` 或 `usm-only.ts` 中新增一条路由，component 指向 `import('/@/views/xxx/index.vue')`。
