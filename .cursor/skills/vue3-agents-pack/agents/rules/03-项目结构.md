# 项目结构（NON-NEGOTIABLE）

## 目录结构

```
src/
├── api/           # 接口请求封装，按模块分目录
├── assets/        # 静态资源
├── components/    # 可复用 UI 组件
├── layout/        # 布局与路由出口
├── router/        # 路由配置（routes 下按 common、usm-only 等拆分）
├── stores/        # Pinia 全局状态，按业务分文件
├── types/         # 全局类型声明
├── utils/         # 工具函数（含 request 等）
├── views/         # 页面视图，与路由一一对应
├── theme/         # 主题、scss 变量、mixins
├── i18n/          # 国际化
├── directive/     # 自定义指令
├── mock/          # 模拟数据（如需要）
└── 根级入口文件（main.ts、App.vue 等）
```

## 结构约束

- 接口请求与封装 → `api/<module>/`，类型可放在 `api` 或 `types`。
- 全局状态 → `stores/`，每业务模块单独文件，禁止在 src 下新建其它 store 目录。
- 页面视图 → `views/`，与 router/routes 中 component 路径一致。
- 通用组件 → `components/`；仅单页使用 → `views/<page>/` 下 components 或同级目录。
- 路由配置 → `router/routes/common.ts`、`router/routes/usm-only.ts` 等，见 06-路由规范。
