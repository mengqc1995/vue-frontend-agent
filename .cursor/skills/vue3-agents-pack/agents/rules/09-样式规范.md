# 样式规范

## 基础原则

- 样式采用 SCSS，使用 scoped；动态 class 用 classnames 或绑定实现。
- 自定义样式必须使用项目主题/变量，禁止硬编码颜色（见下方主题变量）。
- 项目有 `theme/` 目录（scss、变量、mixins），按需引用。

## 主题变量（NON-NEGOTIABLE）

- 主色、文本、背景、边框等使用项目 theme 或 Element Plus 变量（如项目已配置）。
- 确保主题切换时表现一致，不写死色值。

## 禁止内联样式（必须遵守）

- **不允许在模板中使用内联样式**（如 `style`、`:style`）。
- 样式应统一写在 `<style>` 中，并使用 BEM 类名。
- 需要动态样式时，通过 **CSS 变量** 或 **BEM 修饰符类名** 配合 `class` 绑定实现，而非 `:style`。

## BEM 命名规则（必须严格遵守）

所有 CSS/SCSS 类名必须遵循 BEM (Block Element Modifier)：

- **Block（块）**：独立组件/模块的根元素。格式：`block-name`（kebab-case），如 `video-monitoring`、`user-card`。
- **Element（元素）**：块的组成部分。格式：`block-name__element-name`（双下划线 `__`），如 `video-monitoring__header`、`video-monitoring__table-content`。
- **Modifier（修饰符）**：块或元素的状态/变体。格式：`block-name--modifier` 或 `block-name__element-name--modifier`（双连字符 `--`），如 `video-monitoring--active`、`video-monitoring__button--primary`。

### 核心规则

1. 类名使用小写字母和连字符（kebab-case）：`user-card` ✅，`userCard` ❌。
2. 块与元素之间用双下划线 `__`：`block__element` ✅，`block-element` ❌。
3. 修饰符用双连字符 `--`：`block--modifier` ✅，`block-modifier` ❌。
4. 每个组件只有一个 Block；避免嵌套超过 3 层（如 `block__element__sub-element` 为限）。
5. 确定 Block 名称：可根据组件文件名或功能（如 `createRelationship/index.vue` → block 可取 `create-relationship`）。

### SCSS 写法

使用 `&__element` 和 `&--modifier` 嵌套，例如：

```scss
.create-relationship {
  &__header { }
  &__content { }
  &__button--primary { }
}
```

### 禁止用法

- 禁止使用通用类名：`.container`、`.card`、`.table`、`.button` 等（应改为 BEM）。
- 禁止工具类命名：`.mb20`、`.mt10` 等（应合并到相关元素或使用修饰符）。
- 禁止内联样式：`style`、`:style`。

## 生成/审查代码时检查

- 是否未使用内联样式？
- 是否确定了唯一 Block 名称？
- 所有类名是否以 Block 开头？元素是否用 `__`、修饰符是否用 `--`？
- 是否避免使用通用类名与工具类？
- SCSS 是否使用 `&__element` 与 `&--modifier`？
